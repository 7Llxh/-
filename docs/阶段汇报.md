# 汽车尾灯车型识别系统 - 阶段汇报

## 一、项目概述

基于**图像检索**的车型判别系统。流程:车辆检测(YOLOv8s)-> 尾灯定位(部件检测器)-> 特征检索(ResNet50+ArcFace+FAISS)-> 输出**车型 + 年份**。只走尾灯识别:检测到尾灯才检索尾灯库,检测不到输出"未识别到车尾灯"。

## 二、主要功能原理

### 1. 部件检测器（尾灯定位）

- **COCO 预训练 + 微调**:YOLOv8s 先在 COCO 数据集（80类通用物体,30万图）上预训练,学会通用视觉特征;再用人工标注的部件框（8类:taillight/headlight/mirror/window/wheel/plate/grille/exhaust）微调,学会识别车辆部件。
- **作用**:在车辆裁剪图上定位尾灯框（taillight 类）,供尾灯特征提取。
- **迁移学习**:COCO 提供"会看"的基础,标注教它"认部件",100张图即可训出 taillight mAP50=0.984。

### 2. 尾灯库构建（特征检索库）

- **尾灯裁剪**:部件检测器在 rear 车辆图上定位尾灯框 -> 裁尾灯小图 -> 归一化 224×224。
- **特征提取**:ResNet50+ArcFace 将尾灯小图编码成 512 维特征向量（L2 归一化）。
- **ArcFace 度量学习**:训练时用 ArcFace loss（margin=0.5, scale=64）,让同车型尾灯特征靠近、异车型远离。
- **FAISS 索引**:所有尾灯特征向量构建 IndexFlatIP（内积=cosine 相似度）,供快速检索。

### 3. 识别流程

```
输入图 -> 车辆检测(YOLOv8s COCO) -> 部件检测器定位尾灯
  ├─ 有尾灯 -> 尾灯裁剪 -> ResNet50 提512维特征 -> FAISS检索尾灯库 -> 车型+年份
  └─ 无尾灯 -> "未识别到车尾灯"
-> 可视化（结果图 + JSON）
```

- **车型判别**:检索尾灯库 top-K 候选,取 model_series 众数（候选最多的车型）。
- **年份判别**:该车型候选的 year 统计,集中给"约{min}-{max}",分散标"年份不确定"。
- **置信度**:该车型候选的最高 cosine 相似度（0-1）,<0.5 标"未知(置信度低)"。

## 三、本阶段工作

1. **接入 VMMRdb 大规模数据**:28.5 万图 / 9170 个车型年份目录,归并为 1175 个车型系列,剔除图片<20 的稀疏系列,保留 **695 系列**。

2. **车型/年份分离识别**:输出车型(make_model)+ **模糊年份**(年代范围/众数,分散标"年份不确定"),满足"年份可模糊"需求。

3. **部件检测器优化**:
   - 类别 9->8(去冗余 bumper,数据集中)
   - 扩标注多品牌(audi/bentley/bmw 等)提升泛化
   - 主车聚焦(取最大框,排除远车干扰)

4. **标注增强闭环**:check 产漏检队列 -> annotate_tool 按 `n` 跳转补标 -> 重训部件。

5. **小范围高精度验证**:5 车型 × 100 图,尾灯 R@1=**0.754**(较 50 系列×30 图的 0.20 提升 3.7 倍),验证"少车型 + 多图可高精度"。

6. **工具链完善**:一键子集验证、批量识别 + 尾灯框可视化、漏检判读优化(区分后侧方单尾灯)、标注工具导航增强。

## 四、当前成果

| 指标 | 结果 |
|---|---|
| 部件检测器 | 8 类,taillight mAP50=0.984,新车尾灯检测正常 |
| 特征网络(5 车型 100 图) | 尾灯 R@1=0.754 |
| 数据 | VMMRdb 全量(695 系列)就绪 |
| 全流程 | survey -> make -> train -> build -> recognize 跑通 |

**发现**:部件检测器对老车型(70-80 年代)尾灯漏检(形状特殊没见过),新车正常;已接受现状(老车型图少,性价比低)。

## 五、下一步

1. **扩量提精度**:5 -> 10 -> 20 系列(每类全图),逐步提升 R@1
2. **批量评估脚本**:量化 top1 准确率(目前单图验证)
3. **老车型**:暂不处理(性价比低)
